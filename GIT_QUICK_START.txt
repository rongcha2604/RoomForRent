📋 LẦN ĐẦU SETUP (Làm 1 lần duy nhất)
═══════════════════════════════════════════════════════════════

1️⃣  CẤU HÌNH GIT (1 lần duy nhất)
───────────────────────────────────────────────────────────────
git config --global user.name "Tên Của Bạn"
git config --global user.email "email@example.com"


2️⃣  KHỞI TẠO REPOSITORY
───────────────────────────────────────────────────────────────
cd /d "D:\HocTapLTHT\Dự án đã hoàn tất\QuanLyNhaTro-VIP"
git init
git add .
git commit -m "Initial commit: QuanLyNhaTro app"


3️⃣  TẠO REPOSITORY TRÊN GITHUB
───────────────────────────────────────────────────────────────
1. Vào: https://github.com/new
2. Repository name: QuanLy-NhaTro
3. KHÔNG check: README, .gitignore, license
4. Click "Create repository"


4️⃣  KẾT NỐI VÀ PUSH CODE
───────────────────────────────────────────────────────────────
git remote add origin https://github.com/USERNAME/QuanLy-NhaTro.git
git branch -M main
git push -u origin main

(Lưu ý: GitHub sẽ hỏi username/password - dùng Personal Access Token)

═══════════════════════════════════════════════════════════════
🔄 CẬP NHẬT CODE (Làm mỗi khi thay đổi code)
═══════════════════════════════════════════════════════════════

git add .
git commit -m "Mô tả thay đổi"
git push

═══════════════════════════════════════════════════════════════
🔑 TẠO PERSONAL ACCESS TOKEN (Nếu Git hỏi password)
═══════════════════════════════════════════════════════════════

1. GitHub → Settings (avatar góc trên phải)
2. Developer settings → Personal access tokens → Tokens (classic)
3. Generate new token (classic)
4. Check "repo" permission
5. Generate và copy token (chỉ hiện 1 lần!)
6. Dùng token này khi Git hỏi password

═══════════════════════════════════════════════════════════════
❌ XỬ LÝ LỖI THƯỜNG GẶP
═══════════════════════════════════════════════════════════════

Lỗi: "remote origin already exists"
→ git remote remove origin
→ git remote add origin https://github.com/USERNAME/REPO.git

Lỗi: "Permission denied"
→ Dùng Personal Access Token thay vì password

Lỗi: "not a git repository"
→ cd vào đúng thư mục project

Lỗi: "Updates were rejected"
→ git pull origin main --allow-unrelated-histories
→ git push
